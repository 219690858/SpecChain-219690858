# SpecChain — Headspace (EECS 4312)

This repo is my SpecChain project for **Headspace** (Google Play package `com.getsomeheadspace.android`). I take Play Store reviews and run them through **manual**, **automated**, and **hybrid** pipelines to get personas, specs, tests, and metrics.

## What app / data

- **App:** Headspace (meditation + sleep).
- **Raw reviews:** `data/reviews_raw.jsonl` — **1,200** rows (also in `data/dataset_metadata.json`).
- **Cleaned reviews:** `data/reviews_clean.jsonl` — **1,110** rows after we filter/normalize in `02_clean.py`.

Reviews were collected with **google_play_scraper** (see metadata). If `reviews_raw.jsonl` already exists and isn’t empty, `01_collect_or_import.py` won’t wipe it — so re-runs stay stable.

## Where stuff lives

| Folder | What’s in it |
|--------|----------------|
| `data/` | Raw + clean jsonl, metadata, `review_groups_manual.json` / `_auto` / `_hybrid` |
| `personas/` | `personas_manual.json`, `personas_auto.json`, `personas_hybrid.json` |
| `spec/` | `spec_manual.md`, `spec_auto.md`, `spec_hybrid.md` |
| `tests/` | `tests_manual.json`, `tests_auto.json`, `tests_hybrid.json` |
| `metrics/` | `metrics_*.json` + `metrics_summary.json` (compare the 3 pipelines) |
| `prompts/` | `prompt_auto.json` (auto pipeline prompt dump) |
| `reflection/` | `reflection.md` |
| `src/` | All the Python scripts |

**Heads-up:** `run_all.py` only runs the **automated** chain. Manual + hybrid files are stuff I edited / validated separately (`03`, `04`, etc.) — they’re already in the repo for you to read.

## Before you run anything

1. **Python 3** (3.10+ is fine).
2. From the repo root:

   ```bash
   pip install -r requirements.txt
   ```

3. For the LLM steps (`05`, `06`, `07`) you need **Groq**:

   ```bash
   export GROQ_API_KEY="your_key_here"
   ```

   Windows PowerShell: `$env:GROQ_API_KEY="..."`

## Commands (from repo root)

I’m on a Mac so I usually type `python3`; if `python` works for you, use that.

**1. Sanity check — are all the required files there?**

```bash
python3 src/00_validate_repo.py
```

**2. Run the whole automated pipeline end-to-end**

```bash
python3 src/run_all.py
```

That runs, in order: `01` → `02` → `05` → `06` → `07` → `08` (collect/clean, auto groups+personas, auto spec, auto tests, metrics).

**3. Optional — run one step only**

```bash
python3 src/02_clean.py
python3 src/08_metrics.py
```

**4. Compare manual vs auto vs hybrid**

Open `metrics/metrics_summary.json` — numbers for personas, requirements, tests, links, etc.

## One more thing

If you re-run `run_all.py`, it **overwrites** the auto outputs (`review_groups_auto.json`, `personas_auto.json`, `spec_auto.md`, `tests_auto.json`, and the metrics files). Manual + hybrid don’t get regenerated by that script.
